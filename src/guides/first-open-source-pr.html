How to open your first pull request (and not panic)

by Ayush  ·  8 min read

The first PR is the hardest one. Not because it's technically difficult — it's actually straightforward. It's hard because everything is unfamiliar and there's no obvious place to start. This guide fixes that.

By the end of this, you'll have forked a repository, made a change, committed it with a proper sign-off, pushed it to GitHub, and opened a real pull request. Every command is written out. No assumed knowledge.

Before you start: three things to check

Make sure you have these set up first.

  • A GitHub account (free at github.com)
  • Git installed — run git --version in your terminal to check. If you get "command not found," install it from git-scm.com/downloads
  • Your name and email configured in Git:
git config --global user.name "Your Full Name"
git config --global user.email "your@email.com"

Step 1: Fork the repository

A fork is your personal copy of someone else's project. You make all your changes in your fork — nothing in the original repo changes until the maintainer reviews and accepts your pull request.

Go to the repo on GitHub and click the Fork button in the top right. GitHub creates a copy under your account. That's it.

Step 2: Clone your fork to your computer

"Cloning" downloads the repository from GitHub to your local machine. Run this (replace YOUR-USERNAME): 

git clone https://github.com/YOUR-USERNAME/First_Pr_lab.git
cd First_Pr_lab

You should now be inside the project folder.

Step 3: Create a branch

A branch is an isolated workspace. Making changes on a branch means your work doesn't interfere with the main codebase until it's ready and reviewed. 

git checkout -b intro/your-github-username

You're now on a new branch. Any changes you make stay here until you're ready to share them.

Step 4: Make your change

For your intro PR, open CONTRIBUTORS.md in any text editor and add a new row to the table with your name, GitHub username, and LinkedIn link: 

| Your Full Name | [@username](https://github.com/username) | [LinkedIn](https://linkedin.com/in/your-profile) |

Save the file.

Step 5: Commit your change

A commit is a snapshot of your changes with a message describing what you did. The -s flag adds a "Signed-off-by" line — this is called a DCO sign-off, and it's required by this project. 

git add CONTRIBUTORS.md
git commit -s -m "docs: add Your Name to CONTRIBUTORS.md"

To verify the sign-off got added:

git log -1

You should see a Signed-off-by: Your Name <your@email> line at the bottom of the commit.

Step 6: Push your branch

git push origin intro/your-github-username

If GitHub asks for a password: use a Personal Access Token, not your account password. Create one at GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic). Check the repo scope.

Step 7: Open the pull request

Go to your fork on GitHub. You'll see a banner: "Compare and pull request." Click it. Make sure you're targeting the main branch of the original repo.

For intro PRs, the title must be exactly: My First Open Source Contribution

Fill in the PR template and hit submit. Done.

What happens next

A welcome bot will comment within a minute or two. The maintainer will review within a day or two. If there's anything to fix, they'll leave a comment — just make the edit, commit again, and push. The PR updates automatically.

Once it's merged, your name is in the git history of a real open source project. That's your first contribution.

The full version of this walkthrough, including troubleshooting common errors, is in CONTRIBUTING.md. If you get stuck at any step, reach out to Ayush on LinkedIn.
