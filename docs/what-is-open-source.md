# What is Open Source?

Open source software is software where the source code is publicly available. Anyone can read it, copy it, modify it, and contribute back to it. The code is not locked away inside a company - it lives on platforms like GitHub where the entire world can see it.

Most of the software the internet runs on is open source. Linux powers the servers behind almost every major website. Git, the version control tool you use when contributing here, is open source. VS Code, Node.js, Python, Kubernetes - all open source. The list is long.

## Why do people contribute for free?

This is the question everyone asks first. A few honest answers:

Some contributors are paid to do it. Companies that depend on a project often assign engineers to maintain and improve it because its in their interest for the project to stay healthy.

Some contribute to learn. Working on a real codebase with real reviewers teaches you things no tutorial can. You read other people's code, you get feedback on yours, and you ship something that other people actually use.

Some contribute because they found a bug that affects them, fixed it, and shared the fix so nobody else has to deal with it.

Some just like it.

## What you're agreeing to when you contribute

When you submit a pull request, you're saying: "I wrote this, I have the right to contribute it, and I'm okay with it being used under this project's license." That's what the DCO sign-off (`git commit -s`) makes explicit. It's not a legal trap - it's just a clear record.

## The difference between open source and free software

These are related but not the same. "Free software" (from the Free Software Foundation) is about freedom - your right to use, study, modify, and share. "Open source" is a broader umbrella that includes licenses with more restrictions. In practice most projects use the terms interchangeably, but the distinction matters when you're choosing a license for your own project.
